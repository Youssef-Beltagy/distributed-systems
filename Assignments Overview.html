Language

Operating Systems

The operating system used for all assignments is Linux CentOS (cssmpi1h.uwb.edu - cssmpi8h.uwb.edu):

  1. To remotely log into a Linux machine, use ssh and establish a connection to cssmpiNh.uwb.edu where N is 1 through to 8.
    From any of these machines, you can see your home directory (i.e., folder) though NFS.
  2. UW1-320 Lab is our Linux Laboratory where you can use a console of any Linux machines. However, please note that they are not the same as cssmpiNh.uwb.edu (where N = 1 - 8). They are uw1-320-00p.uwb.edu - uw1-320-15p.uwb.edu. Therefore, for your program coding and execution, you still need to log in cssmpiNh.uwb.edu.
  3. To transfer files to the cssmpiNh
    1. From an off-campus Linux machine: use the sftp command and establish a connection to cssmpiNh.uwb.edu.
    2. From a Windows machine: use WinSCP, more information at: http://winscp.net/
    3. From a Mac: use FileZilla, downloadable at https://filezilla-project.org/

The Source for Java Technology

Visit http://www.oracle.com/technetwork/java/index.html . This site includes:
  1. Java Tutorial: helps you run Java programs at Windows, Unix, and Mac, introduces the Java language, and lists essential Java classes.
  2. Java SE Version 8, API documentation: lists all necessary Java API classes. You will frequently visit this page to review what classes are ready to use for your assignment programs.
  3. Java SE : allows you to download the Java SE platform

Java Programming Environments

  • Compile-time errors: The javac compiler points out where your source program has syntax errors. Carefully look at its error messages. You will find out code line numbers that have syntax errors. Open your source code, go to those lines, and fix your bugs.
  • Run-time errors: The java virtual machine points out where your compiled program has run-time errors. Carefully look at its error messages. You will find out your source code's line numbers that have run-time errors. Open your source code, go to those lines, and fix your bugs.
  • Debugging tools: The following three may be helpful to develop your Java program. However, please note that they may not be always useful when you develop multi-threaded programs. (Most assignments are multi-threaded programs)
    1. Jcreator
    2. NetBeansIDE (available at UW1-320)
    3. Eclipse (available at UW1-320)

Assignments
  1. Program 1 (pdf, word): learns how to use Java-based TCP communication through a design of chat client and server program.
  2. Program 2 (pdf, word): exercises how to write a parallel-computing application using MPI Java.
  3. Program 3 (pdf, word): compares RMI and mobile agenets in terms of programmability and performance.
  4. Program 4 (pdf, word): designs and implements a very simple distributed file system. (This year, all students will implement only the file server independently.)
